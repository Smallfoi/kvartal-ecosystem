# STAW Ecosystem — правила совместной работы (Claude + Codex)

Это монорепо экосистемы **STAW**. Над ним **по очереди** работают два ИИ-агента —
**Claude** и **Codex** — и владелец проекта. Вы одна команда: один вектор, один
общий опыт. Главное правило: **ничего не теряется между сменами, и вы не
повторяете ошибок друг друга.**

> Этот файл и `AGENTS.md` — зеркала: одни и те же правила для Claude и Codex.
> Меняешь один — синхронизируй второй.

## Состав экосистемы
- `kvartal-app/` — «Квартал», Flutter + Riverpod. Бег, захват территорий, баллы, клубы.
- `sport_store/` — SportStore, Flutter + Provider. Магазин.
- `САЙТ STAW/` — сайт бренда (статика).
- `backend/` — ОБЩИЙ бэкенд (сейчас FastAPI + SQLite, dev). Единый аккаунт + общие баллы для всех.
- `ECOSYSTEM_API.md` — контракт API, единый источник правды по эндпоинтам для всех трёх продуктов.

Это не три отдельных продукта, а одна система: единый пользователь (SSO по телефону),
единый баланс баллов, общий backend. Баллы зарабатываются в Квартале (бег) и тратятся в Store.

## Единый источник правды о работе (ОБЯЗАТЕЛЬНО к ведению)
- **`docs/WORKLOG.md`** — журнал: *что сделано / что пробовали / что не вышло / что дальше*.
  Читать ПЕРЕД работой (хотя бы последние записи), дополнять ПОСЛЕ.
- **`docs/DECISIONS.md`** — принятые решения и их причины. Не переобсуждать без новой причины.
- **`docs/PITFALLS.md`** — грабли, тупиковые попытки, особенности окружения.
  **Не повторять то, что здесь уже записано как «не работает».**
- **`docs/MODULES.md`** — карта модулей экосистемы (что за модуль, где живёт, статус).
  Меняешь модуль — обнови его статус здесь.

## Перед началом работы
1. `git checkout main && git pull`.
2. Прочитать свежие записи `docs/WORKLOG.md` и весь `docs/PITFALLS.md`.
3. Проверить `git status` и текущую ветку.

## Во время работы
- `main` ЗАЩИЩЁН: прямой push запрещён. Работать в ветке: `feature/*`, `fix/*`, `docs/*`, `chore/*`.
- Все изменения — через **Pull Request**, CI должен быть **зелёным**
  (Flutter analyze/test для обоих приложений + проверка бэкенда).
- Держать **контракты API стабильными** (особенно `auth` / `profile` / `loyalty`) — их используют все приложения.
- **Дизайн/бренд/цвета/шрифты НЕ трогать** — пока ничего не утверждено (директива владельца).
  Сейчас важно одно: чтобы всё работало правильно, как экосистема, без багов. Красота — потом.
- Репозиторий **ПУБЛИЧНЫЙ**: никогда не коммитить секреты/PII (`.mcp.json`, `*.db` уже в `.gitignore`).
- Не двигать большие папки до согласования структуры монорепо.

## После работы
- Дополнить `docs/WORKLOG.md` записью: `## YYYY-MM-DD — <агент> — заголовок` + блоки
  *Сделано / Пробовали — не вышло / Решения / Состояние / Дальше*.
- Новое архитектурное решение → `docs/DECISIONS.md`. Новые грабли → `docs/PITFALLS.md`.
- Влить PR (squash). Убрать временные файлы.

## Окружение и команды (важное)
- ОС: Windows. Оболочки: PowerShell (основная) и git-bash. **Кириллица в PowerShell бьётся** —
  UTF-8-тексты писать через файловый инструмент или Python, не через PowerShell heredoc.
- Бэкенд: `cd backend && PYTHONUNBUFFERED=1 python -m uvicorn main:app --host 0.0.0.0 --port 8000`.
  Бек в dev нестабилен/часто падает — **проверяй `GET /v1/health` перед тестами**.
- Телефон ↔ бек: `adb reverse tcp:8000 tcp:8000` (по USB). На улице связи с dev-беком нет —
  начисления уходят в офлайн-очередь и долетают при возвращении.
- GitHub: репозиторий **`Smallfoi/kvartal-ecosystem`** (public). `gh` лежит по
  `C:\Program Files\GitHub CLI\gh.exe` (в PATH появляется после перезапуска терминала).
