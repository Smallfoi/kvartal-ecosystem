# Шаг 2 — оператор данных и возрастная модель (материал для решения)

Подготовка к шагу 2 «Порядка работ» (`docs/LAUNCH_READINESS.md` §14). Связано с
`docs/STRATEGY_BACKLOG.md` S-16 (возраст), S-08 (аккаунт/безопасность), §2/§8 LR.

> ⚠️ Это **не юридическое заключение**, а разбор вариантов с рекомендациями, чтобы
> владельцу было проще решить. Финальный выбор по налогам, ответственности и
> формулировкам подтверждает профильный юрист (IT/ПДн/e-commerce) и бухгалтер.
> Нормы РФ ниже — общеизвестные ориентиры, требующие проверки на актуальность.

## 1. Юрлицо / оператор данных (кто юридически отвечает)

Экосистема = магазин (перепродажа товаров) + обработка ПДн (телефон, email, GPS,
покупки, баллы) + приём платежей + лояльность. Оператором ПДн и продавцом должно
быть зарегистрированное лицо.

### Варианты
- **Самозанятый (НПД)** — **не подходит**: нельзя перепродажа чужих товаров, нельзя
  наёмные сотрудники, лимит дохода 2,4 млн ₽/год. Для магазина SportStore не годится.
- **ИП** — быстрый/дешёвый старт, упрощённые режимы (УСН «доходы» 6% / «доходы−расходы»,
  возможен патент на часть деятельности), меньше отчётности. **Минус:** отвечает по
  обязательствам **личным имуществом**; сложнее заводить партнёров/инвестиции.
- **ООО** — отдельное юрлицо, **ограниченная ответственность** (в пределах вклада),
  можно несколько учредителей, проще привлекать инвестиции и партнёров (см. идею
  партнёрской программы в `docs/IDEAS.md`). **Минус:** дороже и сложнее
  (бухгалтерия, отчётность, вывод прибыли через дивиденды/налоги).

### Сравнение
| Критерий | ИП | ООО |
|---|---|---|
| Старт (скорость/цена) | проще, дешевле | дороже, дольше |
| Ответственность | личным имуществом | в пределах вклада |
| Партнёры/инвестиции | сложно | легко (доли) |
| Отчётность/бухгалтерия | проще | сложнее |
| Налоговые режимы | УСН, патент | УСН, ОСН |
| Оператор ПДн (152-ФЗ) | да | да |
| Эквайринг/онлайн-касса | да | да |

### Рекомендация
- **Соло-MVP, быстрый запуск, нет партнёров/инвестора сейчас → ИП на УСН.**
- **Есть партнёры/инвестиции или скорый масштаб (партнёрская сеть, команда) → ООО.**
- Допустимый путь: **стартовать как ИП**, перейти на ООО при росте (это потребует
  миграции договоров/оператора ПДн — заложить заранее).
- В любом случае нужно: регистрация как **оператор ПДн** + уведомление РКН (см. §3),
  выбор налогового режима с бухгалтером, расчётный счёт, онлайн-касса (54-ФЗ) к платежам.

## 2. Возрастная модель

### Варианты
- **18+ на старте (рекомендуется).** Простейший правовой режим: не нужен специальный
  детский контур (родительское согласие, усиленная модерация, защита геолокации детей).
  На регистрации — подтверждение возраста; в соглашении — «сервис для лиц 18+».
- **Детский сценарий (<18).** Требует родительского согласия на обработку ПДн ребёнка,
  особой защиты, и **никакой публичной точной геолокации детей** (LR §8). Юридически и
  продуктово тяжелее; для бегового приложения с картой/соцфункциями — высокий риск.

### Контекст РФ (ориентир, проверить с юристом)
- Обработка ПДн несовершеннолетних, как правило, требует согласия законного представителя
  (для младших — полностью, для подростков — по ситуации/виду данных).
- Публикация точной геопозиции ребёнка и соц-функции (поиск рядом, чаты) — зона особого риска.

### Рекомендация
- **Первый публичный релиз — 18+.** Возрастное подтверждение на регистрации + пункт в
  соглашении. Детский режим — отдельно, только после юридической проработки (S-16).
- Влияние на продукт (что заложить): шаг подтверждения возраста при регистрации; настройки
  приватности по умолчанию закрытые (LR §2); геопозиция/маршрут не публичны без действия
  пользователя.

## 3. Роскомнадзор и локализация данных

- **Уведомление об обработке ПДн (152-ФЗ).** При нашей обработке (телефон, email, GPS,
  покупки) уведомление РКН, как правило, **обязательно до начала production-обработки**.
  Указывают: цели, категории данных и субъектов, правовое основание, перечень действий,
  место хранения, меры защиты, привлечённых обработчиков (облако и т.п.).
- **Локализация (242-ФЗ).** ПДн граждан РФ должны храниться в БД, **расположенных в РФ**
  (первичная запись/обновление). → согласуется с D-17 (Yandex Cloud, РФ). Внешние сервисы
  из США (**Google Fonts, CARTO-тайлы, Unsplash** — см. `DATA_INVENTORY.md`) — митигировать:
  бандлить шрифты локально, рассмотреть RF/self-host тайлы, свои фото товаров.
- **Трансграничная передача.** Любая передача за рубеж — отдельно оценить/уведомить.
