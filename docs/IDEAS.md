## Идеи по Runner App

### 💡 Ресурс кроссовок (трекер износа)

**Суть:** Показывать пользователю сколько км осталось у его кроссовок до замены.

**Как работает:**
- Пользователь купил кроссовки в Sport Store → они автоматически регистрируются в Runner App
- Runner App отслеживает каждую пробежку и накапливает километраж именно в этих кроссовках
- При регистрации или в настройках профиля собираем **антропометрические данные**: вес, рост, стиль бега (пронация/супинация)
- По **формуле динамики износа** (км × коэффициент веса × коэффициент стиля бега) считаем реальный остаток ресурса
- Приложение показывает: "Осталось ~230 км из 600 км" — в виде прогресс-бара или счётчика
- Носит **рекомендательный характер** — не жёсткое ограничение, а подсказка
- Когда ресурс на исходе — пуш-уведомление + рекомендация новой модели из Sport Store

**Где интегрировать:** Runner App (основной трекинг) + Sport Store (регистрация покупки + рекомендация замены)

**Связь с экосистемой:** покупка в Store → регистрация в Runner App → трекинг → истощение ресурса → новая покупка в Store

*Статус: идея, требует проработки формулы износа*

### 💡 Набор инструментов бегуна (адаптировать из Simpla)

> Добавлено 2026-06-28. Подсмотрено у конкурента **Simpla** — у них есть набор полезных тренировочных инструментов (см. разбор конкурента).

**Суть:** мини-набор практичных инструментов для бегунов прямо в приложении — для тех, кто реально тренируется и в них нуждается. Берём идею у Simpla и **адаптируем под наш бренд/UX**, не копируя 1:1.

**Какие инструменты:**
- **Калькулятор пульсовых зон** (по возрасту / максимальному пульсу) — зоны Z1–Z5.
- **Интервальный таймер** — настраиваемые интервалы работа/отдых со звуком и вибро.
- **Метроном каденса** — задаёт частоту шагов (шагов/мин) для постановки техники.
- **Конвертер темп ↔ скорость** + расчёт времени / дистанции / темпа.
- **Калькулятор размера кроссовок** (стыкуется со Store — подсказать размер при покупке).

**Зачем:** повышает ценность для продвинутых бегунов (удержание), даёт причину открывать приложение помимо самого забега; часть инструментов (размер кроссовок, каденс) — мостик к Store и к AI-подбору ниже.

**Где интегрировать:** Runner App (Квартал) — раздел «Инструменты»; калькулятор размера — связка со Sport Store.
**Связь с экосистемой:** инструменты → данные о бегуне (каденс, пульсовые зоны, размер) → точнее работают [AI-помощник по подбору](#-ai-помощник-по-подбору-экипировки-не-только-кроссовки) и [трекер износа](#-ресурс-кроссовок-трекер-износа).

*Статус: ✅ СДЕЛАНО (2026-06-29) — все 5 инструментов в Квартале (Профиль → «Инструменты бегуна»), PR #194–#196. Звук/вибро — встроенные (SystemSound + HapticFeedback).*

## Идеи по Sport Store

### 💡 Сравнение товаров (таблица сравнения, как в ДНС)

> Добавлено 2026-06-28. Разобран паттерн ДНС (dns-shop.ru/compare) и лучшие практики.

**Суть:** пользователь добавляет несколько товаров «в сравнение» и видит их рядом по характеристикам — сразу понимает, чем они отличаются и что выбрать. Особенно полезно для технически сложного выбора (кроссовки: назначение, вес, дроп, амортизация, пронация, поверхность).

**Как работает (по разбору ДНС):**
- На карточке товара и в каталоге — кнопка/чекбокс **«Сравнить»** (добавить в сравнение).
- Отдельный экран/страница сравнения: **товары — колонки, характеристики — строки**; сверху фото, название, цена, рейтинг.
- **Тумблер «показать только отличия»** — скрывает одинаковые характеристики, оставляет различающиеся (ключевая фишка ДНС).
- До **~5 товаров** одновременно.
- Для кроссовок строки сравнения: назначение (бег/трейл/зал), вес, дроп (перепад), уровень амортизации, тип пронации, сезон, материал верха, цена, рейтинг.

**Плюсы:** ускоряет выбор, снимает «паралич выбора», повышает конверсию на технически сложных товарах; наглядно вместо листания вкладок.
**Минусы/нюансы:** нужна **единая структурированная схема характеристик** у всех товаров (specs как key-value в `catalog`) — иначе сравнивать нечего; на мобильном — горизонтальный скролл/адаптив.

**Где интегрировать:** Sport Store (приложение) **+ Сайт** — одна модель данных и один бэкенд `catalog`.
**Связь с экосистемой:** единый `catalog` хранит атрибуты товара структурированно → и сайт, и приложение строят сравнение из одних данных. **Предусловие общее с идеей AI-подбора ниже** — нормализованные характеристики товаров.

*Статус: идея; предусловие — нормализованная схема характеристик в `catalog`.*

### 💡 AI-помощник по подбору экипировки (не только кроссовки)

> Добавлено 2026-06-28.

**Суть:** ИИ-ассистент-консультант. Пользователь отвечает на вопросы, а ИИ **задаёт встречные наводящие вопросы** (живой диалог), **строит уникальный портрет клиента** и подбирает **кроссовки + полную экипировку** по **уровню подготовки, специализации и ценовой категории**.

**Как работает:**
- **Диалоговый подбор:** ИИ уточняет — уровень (новичок / любитель / продвинутый), **специализацию** (дисциплина/дистанция: спринт, 5–10 км, полумарафон/марафон, трейл), цель (похудеть / здоровье / результат), частоту, поверхность (асфальт / трейл / зал), **ценовую категорию / бюджет**, особенности (пронация, травмы, размер).
- **Уникальный портрет клиента:** из ответов (а со временем — из данных бега в «Квартале» и инструментов: каденс, пульсовые зоны, размер) собирается **сохраняемый профиль бегуна**, который переиспользуется и уточняется при следующих подборах.
- **Наводящие вопросы**, если данных мало — как живой консультант в магазине.
- На выходе — подборка **под уровень и бюджет:** новичку → стартовые бюджетные модели; продвинутому → продвинутые модели; **плюс сопутствующая экипировка комплектом** (одежда, аксессуары), а не только пара кроссовок.
- Подбор строго из **реального каталога** (товары / наличие / цена из `catalog`) — без выдуманных товаров.

**Плюсы:** снимает барьер «не знаю, что выбрать»; повышает средний чек (комплект, а не одна пара); персонализация как у топ-брендов; вовлечение.
**Нюансы/подводные камни:** нужен каталог со **структурированными атрибутами** (та же база, что и для сравнения выше); ИИ подбирает только из реального каталога (RAG/функции по `catalog`, без галлюцинаций); стоимость токенов; модерация ответов; начать можно с правил + LLM поверх. ИИ — на актуальной модели (например, Claude).

**Где интегрировать:** Sport Store **+ Сайт** (диалог), данные из `catalog`; **уровень подготовки может подтягиваться из Runner App** (история бега, км/опыт) — кросс-связь.
**Связь с экосистемой:** уровень бегуна из «Квартала» → точнее подбор в Store → покупка → регистрация кроссовок в Runner App (стыкуется с идеей [трекера износа кроссовок](#-ресурс-кроссовок-трекер-износа)).

*Статус: идея; предусловие — структурированный каталог; запускать после выстраивания каталога/бэкенда.*

Сейчас sport_store = «SPORT STORE», чёрно-белый, шрифт Oswald. Общий бренд экосистемы — **STAW / 世拓** (светлый running/lifestyle бренд). Чтобы все три продукта (Квартал + Store + Сайт) шли одним вектором, выровнять Store по бренд-токенам сайта (`САЙТ STAW/BRAND_CONTEXT.md` + `styles.css`):

- **Название:** SPORT STORE → **STAW** (splash, app bar, онбординг, auth).
- **Цвета (из styles.css сайта):** bg молочный `#f4f1ea`, paper `#fffdf8`, текст/тёмные секции графит `#20252b` (вместо чистого чёрного), акцент голубой `#57bcd8`, сигнал лайм `#dff45f`, muted `#6f7278`, soft `#e7eef0`.
- **Шрифты:** заголовки **Unbounded**, текст **Manrope** (заменить Oswald). Оба в google_fonts.
- **Голос/слоганы:** «Start from motion», «Не ждать выходных. Начинать движение сейчас», «run, train, move daily». Категории бренда: EVERYDAY / RUN / TRAIL / RACE.

Фичи/логику НЕ трогаем — только тема, тексты, нейминг. Это и есть «один вектор» без кардинальных изменений.

## Идеи по Маркетингу и Росту

### 💡 Партнёрская программа для сторонних магазинов (масштабирование)

> ⏳ **Статус: на будущее — реализовывать после полного запуска экосистемы**

**Суть:** Подключать сторонние магазины (спортпит, физио-центры, кафе здорового питания и др.) к экосистеме Кварталов как партнёров.

**Механика:**
- Территория рядом с магазином партнёра = физический трафик бегунов
- Бегуны тратят баллы Кварталов у партнёра как скидку
- Комиссия только за реального клиента (pay-for-performance), не за показы

**Почему это лучше конкурентов:**
- Трафик создаёт игровая механика, а не реклама — люди приходят сами
- Аудитория квалифицирована (активные, спортивные, платёжеспособные)
- Физический трафик в оффлайн-точку — то чего не даёт ни один цифровой канал

**Запуск:** первым партнёрам — бесплатный пробный период 1–2 месяца, затем комиссия

*Реализовывать когда: проект полностью работает, есть активная база пользователей, начинается масштабирование*
