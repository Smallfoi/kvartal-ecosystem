# MODULES — карта модулей экосистемы STAW

Экосистема разбита на **модули** — самостоятельные куски функциональности. Это
помогает Claude и Codex (и владельцу) сразу понимать: что за модуль трогаем, где он
живёт (какое приложение + бэкенд) и в каком он статусе.

**Правило:** меняешь модуль — обнови его **Статус** здесь и запись в `docs/WORKLOG.md`.
Контракт API по модулям — в `ECOSYSTEM_API.md`; решения — в `docs/DECISIONS.md`.

**Легенда статусов:** ✅ готово · 🟡 частично / локально (ждёт бэк) · 🔴 план (ещё нет)

## Принцип «что строим на бэке»
Бэкенд — для данных, которые должны быть **общими и настоящими** (один аккаунт, общие
баллы, клубы, скоро — пробежки и территории). Чисто визуальное/локальное (навигация,
анимации, настройки отображения) бэкенд не требует. Приложение = интерфейс + кэш,
«правда» — на сервере. См. [[project-ecosystem-context]] и `docs/DECISIONS.md`.
