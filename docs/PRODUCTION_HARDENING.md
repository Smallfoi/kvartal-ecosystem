## 🔴 Безопасность — P0
- ⬜🔑 **Секреты из env, не дефолты.** `JWT_SECRET` и `DJANGO_SECRET_KEY` = `dev-secret-change-in-prod` (в коде). Подделка токена любого юзера. DB-пароль `kvartal`. → сильные секреты в secret-store.
- 🟡 **DEBUG=False в проде.** Через env (`DJANGO_DEBUG=0`) + `DJANGO_ALLOWED_HOSTS`. Прод-secure-заголовки (HSTS/SSL-redirect/secure-cookies) уже добавлены за `not DEBUG`. Осталось задать env в проде.
- ⬜🔑 **Настоящий вход (SMS-OTP).** Код `1234` зашит — вход под любым телефоном. → smsc.ru, одноразовые коды со сроком, лимит попыток/запросов.
- ✅ **Rate-limiting.** DRF-throttling: по JWT (`user` 300/мин) + по IP для анонимных (`anon` 120/мин) + жёстко на `/auth` (`auth` 20/мин, анти-брутфорс). _Прод: общий счётчик в Redis (D-07) для нескольких воркеров._
- 🟡 **HTTPS везде.** Secure-заголовки готовы (P0 выше); нужен прод-TLS (домены/сертификаты) и cleartext в апках только для прод-доменов.
- ⬜🔑 **Админка:** пароль `staw-admin-2026` (в доках) → сменить; ограничить `/admin` по IP/VPN; 2FA.
- ✅ **Лимит payload** (точки маршрута ≤ 20000) — анти-DoS на PostGIS.
