## A. Персональные данные (что собираем и где хранится)

Источник правды — общий Django-бэкенд (PostgreSQL/PostGIS). Ниже — таблицы и поля.

| Категория | Поля | Где (модель / хранилище) | Продукты | Чувствительность |
|---|---|---|---|---|
| Идентификация аккаунта | `name`, `email`, `phone`, `city`, `avatar_path`, `provider`, `password_hash` | `accounts.User` (Postgres) | Все (единый SSO) | **Высокая**: телефон/email — прямые идентификаторы |
| Баллы/лояльность | `user_id`, `amount`, `source`, `description`, `order_id`, `run_id` | `loyalty.Transaction` | Квартал + Store | Средняя (поведение, квази-финансы) |
| Заказы | `user_id`, `order_id`, `total`, `status`, `points_redeemed`, `payload` (JSON) | `orders.Order` | Store | **Высокая**: в `payload` — имя, **телефон**, тип/адрес доставки из checkout |
| Клубы (соц-граф) | `name`, `city`, `owner_id`, `user_id`, `role`, `status` | `clubs.Club/Member/JoinRequest` | Квартал | Средняя (связи между людьми) |
| Территории (гео) | захваченные зоны, `MultiPolygon`, привязка к пользователю | `territories` (PostGIS) | Квартал | **Высокая**: где человек бегает → косвенно дом/режим |
| Кроссовки/износ | `user_id`, `product_id`, `model`, `total_km`, `max_km`, `applied_runs` | `shoes` | Квартал + Store | Средняя (поведение; **не медданные**, см. §11) |
| Уведомления | `user_id`, `title`, `body`, `order_id`, `read` | `notifications` | Все | Низкая (но содержит ссылки на заказы) |
| Каталог | товары/категории/баннеры | `catalog` | Store + сайт | Нет PII |

### Геоданные (GPS) — отдельно (§2 — самое чувствительное)
- **Сырой GPS-трек пробежки** (список точек + дистанция) пишется **локально на устройстве**
  (`SharedPreferences`, ключ активного забега; история пробежек — локально). На сервер уходят
  **производные**: захваченные территории (полигоны) + дистанция/баллы. Это снижает риск, но:
  - [ ] ограничить срок хранения сырых точек на устройстве (§2);
  - [ ] при будущей серверной загрузке трека — разделить идентификацию и геоданные (связь по `user_id`);
  - [ ] privacy by design: скрывать начало/конец маршрута у дома, не показывать точное положение другим по умолчанию.

### Локальные хранилища приложений
- **Квартал**: JWT-токен (`kvartal.auth.token.v1`) и активный забег — в `SharedPreferences`.
  Зависимость `flutter_secure_storage` подключена, но токен лежит в обычных prefs.
  - [ ] **Безопасность (S-08):** перенести JWT в `flutter_secure_storage`/Keychain/Keystore.
- **SportStore**: корзина, избранное, JWT, кэш профиля, уведомления — в `SharedPreferences` (кэш, источник правды — бэк).

### Разрешения устройства (привязать к согласиям §3)
- Геолокация (`geolocator`) — Квартал, запрашивать при старте пробежки с объяснением цели.
- Камера (`mobile_scanner`) — Квартал, скан QR вступления в клуб.
- Уведомления (`permission_handler`) — Квартал; фоновые — для трекинга пробежки.
- Доступ к фото (`image_picker`) — SportStore/аватар.
