# DECISIONS — принятые решения по экосистеме STAW

Короткие записи о принятых решениях и их причинах, чтобы Claude и Codex не
переобсуждали одно и то же. Менять решение можно — но с новой записью и причиной.

Формат: `## D-NN — заголовок` + Контекст / Решение / Причина / Статус.

---

## D-08 — Единый источник правды о работе = docs/WORKLOG.md
**Решение:** журнал `docs/WORKLOG.md` + `DECISIONS.md` + `PITFALLS.md`; правила в `CLAUDE.md`/`AGENTS.md`.
**Причина:** Claude и Codex работают по очереди — нужна общая память, чтобы не терять контекст и не повторять ошибки.
**Статус:** активно (2026-06-14).

## D-07 — Docker — нужен, но позже и поэтапно
**Решение:** Docker Compose для локальной инфраструктуры (Django, PostgreSQL/PostGIS, Redis, Celery worker/beat)
вводим ПОСЛЕ старта Django, не на первом шаге. Порядок: GitHub/монорепо → Django scaffold → Postgres/PostGIS → Docker → Redis/Celery.
**Причина:** консистентное окружение для обоих агентов и деплоя; но не блокировать текущие шаги. (Решение Codex.)
**Статус:** запланировано.

## D-06 — Backend мигрируем на Django + DRF, но постепенно
**Решение:** целевой бэкенд — Django + DRF + PostgreSQL/PostGIS. Django поднимать РЯДОМ с FastAPI,
сперва повторить контракт (`auth/phone/verify`, `auth/me`, `profile`, `loyalty`), затем переключать приложения. FastAPI не удалять до готовности Django.
**Причина:** нужна админка, удобная ORM, PostGIS для территорий, масштаб. Но FastAPI уже работает — спешить нельзя.
**Статус:** запланировано (Phase 2+). Детали: `kvartal-app/CLAUDE_HANDOFF_GITHUB_DJANGO_2026-06-13.md`.

## D-05 — Репозиторий публичный
**Решение:** `Smallfoi/kvartal-ecosystem` — PUBLIC.
**Причина:** branch protection / rulesets на приватном репо требуют GitHub Pro (403). Публичный даёт защиту бесплатно.
**Последствие:** НИКОГДА не коммитить секреты/PII. История проверена на секреты перед публикацией — чисто.
**Статус:** активно. (Можно пересмотреть, если купят Pro.)

## D-04 — Рабочий процесс: ветка + PR + зелёный CI, main защищён
**Решение:** прямой push в `main` запрещён (ruleset `protect-main`); изменения через PR, обязательны проверки CI.
**Причина:** два агента + владелец правят одни файлы — без этого затирают друг друга и ломают main.
**Статус:** активно.

## D-03 — Монорепо без перемещения папок
**Решение:** `git init` в корне `D:\MyProjectsCLAUDE`; папки проектов на местах (`kvartal-app/`, `sport_store/`, `САЙТ STAW/`, `backend/`).
**Причина:** монорепо как требует стратегия, но без рискованных перемещений до согласования структуры.
**Статус:** активно.

## D-02 — SSO по телефону, единый аккаунт и общий баланс
**Решение:** один `user_id` на телефон; вход по телефону (dev-код `1234`) в Квартале и Store ведёт в один аккаунт;
баллы — общий счёт (`/loyalty/*`). Бек — источник правды, локальное хранилище приложений — кэш.
**Причина:** ядро смысла экосистемы — единый пользователь и баллы, заработанные в Квартале, тратятся в Store.
**Статус:** активно, доказано на устройстве.

## D-01 — Дизайн/бренд пока не трогаем
**Решение:** не менять цвета/шрифты/брендинг/вёрстку. Только структура, корректность, обмен данными.
**Причина:** дизайн не утверждён, может кардинально поменяться. Сначала — чтобы всё работало без багов.
**Статус:** активно (директива владельца).
