## Что пробовали

- Сначала сайт был обычной витриной магазина.
- Затем был взят Apple MacBook Air как референс по премиальной подаче: крупные сцены, плавность, product-story.
- Была тёмная технологичная версия сайта.
- Пользователь сказал, что сайт слишком тёмный; сайт переведён в более светлую спортивную палитру.
- Пользователь сказал, что кириллица выглядит дешево; системные шрифты заменены на Manrope + Unbounded.
- Пользователь просил найти фото в Taobao; использовались открытые Taobao/Tmall-зеркала и прямые публичные изображения.

## Что не получилось / ограничения

- Прямой обход антибота, логина или ограничений Taobao не выполнялся и не должен выполняться.
- Можно использовать только публично доступные страницы, зеркала и прямые изображения, которые открываются без авторизации.
- Встроенный браузер Codex несколько раз не запускался с ошибкой:

`windows sandbox failed: spawn setup refresh`

Поэтому полноценная визуальная проверка через in-app browser пока не была выполнена.

- Официальный сайт `stawrunning.com` через PowerShell возвращал ошибку `402 Payment Required`, поэтому основной сбор фото шёл через открытые товарные источники.

## Текущие важные решения

- На сайте писать бренд как **STAW**.
- Не использовать `$TAW` в публичном интерфейсе.
- Не писать в публичном тексте, что это китайский бренд. Это внутренний контекст, но не маркетинговый акцент сайта.
- Подача бренда: современный спортивный бренд для бега, тренировки, города и движения каждый день.
- Сайт должен выглядеть светлее, чище, дороже, без дешёвого маркетплейс-ощущения.
- Фотографии пользователя в будущем складывать в `Референсы/Фото товаров`.
- Видео пользователя в будущем складывать в `Референсы/Видео`.

## Единая экосистема проектов

Этот проект не является отдельным продуктом. Он является частью общей экосистемы:

- `bugun-app` / «Квартал» — мобильное приложение для бегунов: захват территорий, баллы, клубы, соревнования.
- `САЙТ STAW` — сайт бренда/магазина, который далее будет переименован под общий бренд.
- `sport_store` — мобильное приложение спортивного магазина.

Перед началом работы обязательно прочитать `RECOMMENDATION.md`. Это общее рекомендательное письмо для всех трёх проектов: в нём собраны выводы по Nike, Adidas, Puma, Gymshark, Under Armour и описана стратегия общей экосистемы, а не изолированных приложений.

Общий смысл: приложения и сайт должны развиваться как связанная система — единый бренд, единые пользователи, заказы, лояльность, баллы, уведомления, контент и будущая backend/admin архитектура.
