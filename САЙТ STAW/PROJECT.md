# Сайт для официального магазина STAW

## Цель

Создать официальный сайт-магазин для китайского бренда STAW / 世拓: презентация бренда, философии, каталога товаров, корзины и подготовка к дальнейшей интеграции с backend/CMS.

## Текущий формат

Статический frontend-прототип:

- HTML
- CSS
- Vanilla JavaScript

## Основные разделы

- Главный экран с брендом STAW
- Анимационная бренд-сцена
- Философия Start Weekend / движение как стиль жизни
- Сцены Urban / Sportswear / Outdoor
- Product-story о функциональной одежде
- Каталог с фильтрами: бег, outdoor, daily
- Демо-корзина

## Референсы

Папка `Референсы/` предназначена для материалов пользователя:

- `Референсы/Фото товаров/` - фотографии реальных товаров
- `Референсы/Видео/` - будущие видео для анимации сайта
- `Референсы/Образцы/` - образцы, логотипы, скриншоты, дополнительные материалы

## Бренд-контекст

Всё исследование по бренду хранится в `BRAND_CONTEXT.md`: философия, стиль, аудитория, сильные и слабые стороны, источники и правила применения на сайте.

## Единая экосистема проектов

Этот проект не является отдельным продуктом. Он является частью общей экосистемы:

- `bugun-app` / «Квартал» — мобильное приложение для бегунов: захват территорий, баллы, клубы, соревнования.
- `САЙТ STAW` — сайт бренда/магазина, который далее будет переименован под общий бренд.
- `sport_store` — мобильное приложение спортивного магазина.

Перед началом работы обязательно прочитать `RECOMMENDATION.md`. Это общее рекомендательное письмо для всех трёх проектов: в нём собраны выводы по Nike, Adidas, Puma, Gymshark, Under Armour и описана стратегия общей экосистемы, а не изолированных приложений.

Общий смысл: приложения и сайт должны развиваться как связанная система — единый бренд, единые пользователи, заказы, лояльность, баллы, уведомления, контент и будущая backend/admin архитектура.
